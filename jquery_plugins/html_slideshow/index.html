HTML Slideshow

Table Of Contents

Each slide requires Chapter and Slide Title elements.
These values are used to build the Table Of Contents and Persistent slide URLs.
We create a dynamic ID from these values, so the raw HTML should not contain IDs on the slides.
(if they do they will get overwritten)
<article>
    <h1> HTML Slideshow </h1>     // chapter 
    <h2> Table Of Contents </h2>  // slide title 
    ...
<article>
Each new value in the chapter tag will display a chapter header in the Table of Contents.

HTML Slideshow

Element Types

The default slide element is article, but can be changed to a class or different tag. 
$('#slideshow').buildSlideshow({
    element_slide: '.slide'
});
The default Chapter and Title elements are h1 & h2, but can also be changed. 
$('#slideshow').buildSlideshow({
    element_slideChapter: '.chapter'
    element_slideTitle: '.title'
});

Persistent History with localStorage

As a fallback, if a user is returning to the site from a static URL, she will return to the last viewed slide.
This is done by setting a localStorage variable of the currently active slide.
You can set the localStorage key that will save the current slide value.
$('#slideshow').buildSlideshow({
    localStorageKey: 'demoSlideshowCurrentSlide'
});
