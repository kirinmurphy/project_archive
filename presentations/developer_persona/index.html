Major Challenges of the growing web

  • Browser Inconsistencies: Microsoft vs. Open Source & the WC3
  • Code maintainability: Too many cooks make spaghetti code.
  • Too much infomation: Early on, twitter would crash all the time
  • Numerous Devices: mobile, tablet, desktop, touchscreen, retina
  • Numerous Clients: web browser, mobile app, 3rd party software
  • Waterfall / Desktop software product development: Too many practices from physical software carried over to online software.
  • Security: still a huge struggle...

The Modern Web

Conventions / Practices

Web Standards

Browser makers attempt to follow a common standards defined by the W3C so web pages work the same everywhere.

Separation of Concerns / Single Responsibility Principle

A piece of code should do one thing well, and rely on other code to do other things well.

DRY (Don't Repeat Yourself)

Common operations should happen in one place.

Semantics

Code should be self describing and human readable.

Agile Development (Scrum, Kanban, Lean)

Iterative development practices built to adapt to changing demands and quick release cycles.

Big Words

Language

Base syntax for executing operations and manipulating data.
Javascript, Ruby, Java, C#, Scala

Platform/Environment

Engine that runs an application and manages frameworks and libraries (collectively the "packages") it uses.
Node.js, Java Virtual Machine, .NET, Web Browsers, IOS, Android

Library

Loose collection of tools that can be used individually or together for similar sets of tasks.
jQuery, Underscore, Bootstrap

Framework

Suite of tools meant to be used in unison to manage a set of interrelated tasks and data.
Angular, React, Rails, Sinatra, Express.js, PlayFramework

Stack

Collection of platforms/frameworks/libraries and databases used for a single app.
MEAN stack: MongoDB, Express Server, Angular.js client, Node.js platform
My current stack: PlayFramework in Scala, JVM platform, SQL & Redis databases, Angular.js client

The Modern Web

REST · AJAX · JSON · Single Page Apps

REST

Commands used over HTTP calls to submit changes between computers. Used for both Server > Client and Server > Server communications. Primary commands: GET, POST, DELETE
"Press 1 for Sales, 2 for Customer Support"

AJAX

Not a language. Feature of Javascript allowing content to be sent without reloading the page. Uses REST commands to make REQUESTS and receive RESPONSES.

JSON

Human readable data format used universally for sharing content between cleints and servers.

Single Page Web Apps (SPA, OPA)

Web applications that do everything without reloading the page.
Soundcloud - browse different music pages without the music stopping.

The Modern Web

Models and Views

MVC / MVVM / MVW / MV*

Frameworks that separate code for data (model), presentation (view), and other elements to be more flexible/reusable. Promotes the single responsibility principle.
Rails, Scala, most JS frontend frameworks

The Modern Web

APIs · Micro Services

APIs (Application Programming Interface)

A format created for allowing 3rd parties to access resources and content. Web APIs also use HTTP and REST to accept REQUESTS and returns responses usually in JSON. AJAX in a browser will often connect to an API.
Twitter Feed Widget, Facebook comments widget, Google Maps widget

Micro Services

Instead of one big application, we make smaller focused elements that can be maintained separately and used interchangably across projects. One site might actually be dozens of tiny little apps that talk to each other through their APIs.
Different services in an app might include Search, Enrollment, Validation, Analytics, Emails/Notifications, etc.

The Modern Web

Mobile Apps · Hybrid Apps

(Native) Mobile Apps

Built on native platforms. App is stored on your device (not the web), but the data is accessed from the sameAPIs that also serve the website versions.
IOS: Swift, Android: Java

Hybrid Mobile Apps

Build mobile apps with HTML/CSS/JS. Common code can be shared across web and mobile versions.
Ionic, Cordova (Phone Gap)

The Modern Web

UI Design · Responsive Design · Mobile First

Responsive Design

Feature of CSS that allows you to set different rules based on different devices and screen sizes.
Shameless plug: truebluedining.com

Pre-processors

Formatters that provide additional features not (yet) available in native CSS. Files get processed into native CSS readable by a browser.
Less, Sass, PostCSS

Bootstrap

Popular library of commonly used CSS conventions and widgets. Takes some of the complications away from native CSS to make it easier to use for varied skill levels.

Material Design

Google's somewhat new mobile design spec focused on clean, consistent design patterns and responsive interactions.

Image Alterntives

Image files are massive, often causing performance issues (particulary in mobile). There are numerous alternatives to limit the amout of images needed.
CSS3 Shapes, Web Fonts, Icon Fonts, SVG

Screening Candidates

Resume Red Flags

Too many technologies

Laundry list of technologies with no visible preference or focus indicates possible lack of personal motivation or marginal experience with specific tools.

Outdated languages and tools

  • DHTML, XHTML: Obsolete versions of HTML
  • XML, XSLT: Virtually obsolete data format replaced by JSON
  • 3rd party multimedia plugins: (Flash, Silverlight) Mostly replaced by native javascript and HTML5 (Audio, Video and Canvas)

Still active technologies losing steam

  • JQUERY ONLY: Still a tool in the toolbox, but shouldn't be the only/main one. Framworks with Data-Binding have eliminated a large chunk of jQuery's necessity.
  • Backbone, Ember, Knockout: Earlier Javascript frameworks that have lost their competitive edge to more modern frameworks like Angular and React. Not a terrible skillset, but probably shouldn't be used for new projects.
  • SOAP: Format for computers to talk to earch other. REST is now the industry standard.
